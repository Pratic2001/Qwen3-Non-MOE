# --- Recommended: crawl4ai HTML backend ---
# Swaps the httpx+trafilatura HTML path for a real (Playwright-driven)
# headless-browser fetch + LLM-oriented markdown extraction. Handles
# JS-rendered/SPA pages the plain httpx fetch can't see at all (no content
# in the raw response to extract from), and its pruning content filter
# strips nav/ad/boilerplate more aggressively than trafilatura alone. Also
# powers the new deep_crawl tool (multi-page same-domain crawling in one
# call). Missing this dependency doesn't break anything -- server.py's
# SCRAPER_HTML_BACKEND defaults to "auto", which falls back to
# httpx+trafilatura per-URL if crawl4ai isn't installed; set it to "httpx"
# to disable crawl4ai entirely, or "crawl4ai" to require it.
crawl4ai>=0.6
# After `pip install crawl4ai`, it needs a one-time browser + OS-deps setup
# (equivalent to `playwright install --with-deps chromium`):
#   crawl4ai-setup
#   crawl4ai-doctor      # sanity-checks the install before a real run

# --- Optional: video/audio transcription ---
# Caption/subtitle extraction needs only yt-dlp. Local ASR transcription
# (used when no captions exist) additionally needs faster-whisper, plus the
# system binary ffmpeg on PATH.
yt-dlp>=2024.1
faster-whisper>=1.0
# yt-dlp pulls in `requests` transitively. An old `requests` paired with a
# newer urllib3/charset-normalizer prints a noisy (harmless) "doesn't match
# a supported version" RequestsDependencyWarning on every run -- pinning a
# current `requests` here avoids that.
requests>=2.32
# Recommended for full-fidelity YouTube extraction: yt-dlp needs a JS
# runtime to derive signatures for some formats; without one it logs a
# warning (now silenced, see extractors.py) and captions/metadata still
# work, but some audio formats used by the ASR-fallback path may be
# missing. Install deno (https://deno.land) or Node.js, then point yt-dlp
# at it with the YTDLP_JS_RUNTIME env var, e.g. YTDLP_JS_RUNTIME=deno or
# YTDLP_JS_RUNTIME=node:/path/to/node. Not a pip package, so not listed
# below -- install it via your OS package manager / deno's installer.

# --- Optional but recommended once a run exceeds ~10k-100k docs/category ---
